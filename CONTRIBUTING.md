# Contributing

This repository is managed as a structured training project.

## Contribution Principles

- Keep content practical and production-oriented.
- Follow the style guide in `docs/STYLE_GUIDE.md`.
- Prefer clear explanations over exhaustive reference material.
- Use examples that can be demonstrated in a real training session.

## Module Contributions

When adding or updating a module, include:

- Documentation in `module.md`
- A module-level `README.md`
- Slides in `slides/`
- Editable diagrams in `diagrams/`
- Exercises and labs in `labs/`
- Examples in `examples/`
- Quiz questions in `quiz.md`
- References in `references.md`

## Commit Style

Suggested commit prefixes:

- `docs:` documentation changes
- `slides:` presentation material
- `diagrams:` diagrams and visual assets
- `labs:` labs and exercises
- `examples:` sample configuration or code
- `chore:` repository maintenance
