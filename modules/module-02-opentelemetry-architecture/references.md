# Official references

These references support the OpenTelemetry architecture concepts covered in this module.

## OpenTelemetry architecture

- [OpenTelemetry Documentation](https://opentelemetry.io/docs/) - Main project documentation for concepts, instrumentation and deployment guidance.
- [OpenTelemetry Specification](https://opentelemetry.io/docs/specs/otel/) - Normative specification for APIs, SDKs, signals and semantic conventions.
- [OpenTelemetry Protocol Specification](https://opentelemetry.io/docs/specs/otlp/) - Official OTLP documentation for telemetry transport.

## Collector

- [OpenTelemetry Collector](https://opentelemetry.io/docs/collector/) - Official Collector documentation, including receivers, processors, exporters and deployment concepts.
- [Collector Configuration](https://opentelemetry.io/docs/collector/configuration/) - Official guidance for defining Collector pipelines.

## Context and correlation

- [OpenTelemetry Context Propagation](https://opentelemetry.io/docs/concepts/context-propagation/) - Conceptual reference for carrying request context across boundaries.
- [W3C Trace Context](https://www.w3.org/TR/trace-context/) - Standard HTTP trace context propagation format.

## How to use these references

Use the specification when exact behavior matters. Use the Collector documentation when preparing labs or implementation examples. Use the context propagation references when explaining why architecture diagrams must include service boundaries and request identity.
