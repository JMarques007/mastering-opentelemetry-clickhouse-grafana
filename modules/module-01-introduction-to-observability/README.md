# Module 01 - Introduction to Observability

This module introduces observability as a production engineering discipline. It explains why modern systems need more than dashboards and isolated logs, then builds the foundation for logs, metrics, traces, correlation and investigation workflows.

## Learning objectives

After completing this module, learners should be able to:

- Explain the difference between monitoring and observability.
- Describe the role of logs, metrics and traces.
- Understand why correlation is essential during incidents.
- Follow a simple investigation workflow from symptom to evidence.
- Recognize how OpenTelemetry supports observability.
