# Official references

These references support the concepts introduced in this module. Prefer them when expanding the course material or preparing instructor notes.

## OpenTelemetry

- [OpenTelemetry Documentation](https://opentelemetry.io/docs/) - Project documentation for instrumentation, collection and telemetry concepts.
- [OpenTelemetry Specification](https://opentelemetry.io/docs/specs/otel/) - Normative specification for signals, APIs, SDK behavior and semantic conventions.

## Trace context and correlation

- [W3C Trace Context](https://www.w3.org/TR/trace-context/) - Standard HTTP headers for propagating trace identity across service boundaries.
- [OpenTelemetry Context](https://opentelemetry.io/docs/concepts/context-propagation/) - Conceptual documentation for carrying request context through instrumented systems.

## Observability platforms

- [Grafana Documentation](https://grafana.com/docs/) - Official documentation for Grafana dashboards, visualization and observability workflows.
- [Grafana Observability](https://grafana.com/docs/grafana-cloud/monitor-infrastructure/) - Grafana guidance for monitoring infrastructure and service telemetry.

## How to use these references

Use the OpenTelemetry documentation to explain vendor-neutral telemetry concepts. Use the W3C Trace Context reference when teaching correlation and propagation. Use the Grafana documentation when showing how telemetry evidence becomes dashboards, exploration views and operational workflows.
