# Exercise - Checkout error-rate alert

## Goal

Design an actionable alert for elevated checkout error rate.

## Instructions

Define:

- signal and query intent;
- threshold;
- evaluation window;
- severity;
- owner;
- routing target;
- dashboard link;
- trace or log starting point;
- first runbook action;
- safe test method.

## Expected outcome

A strong alert describes user impact, has a clear owner and gives the responder enough context to start investigation. It should not merely say that a number crossed a threshold.

## Instructor notes

Ask learners what the responder should do in the first five minutes. If there is no answer, the alert needs a better runbook or may not be worth paging someone.
