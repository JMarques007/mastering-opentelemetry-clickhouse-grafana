# Roadmap

This roadmap is organized into one-day sprints. Each sprint should produce a usable increment of the course.

## Sprint Plan

| Sprint | Focus | Outcome |
|---:|---|---|
| 01 | Repository Foundation | Project structure, README, roadmap, style guide and module template |
| 02 | Module 2 – OpenTelemetry Architecture | Complete 1-hour training module |
| 03 | Module 3 – OpenTelemetry Collector | Collector documentation, slides, diagrams and lab |
| 04 | Module 4 – Logs | Logs module |
| 05 | Module 5 – Metrics | Metrics module |
| 06 | Module 6 – Traces | Traces module |
| 07 | Module 7 – Context Propagation | Context propagation module |
| 08 | Module 8 – Instrumentation | Instrumentation module |
| 09 | Module 9 – ClickHouse | ClickHouse module |
| 10 | Module 10 – Grafana | Grafana module |
| 11 | Module 11 – Dashboards | Dashboard design module |
| 12 | Module 12 – Alerting | Alerting module |
| 13 | Module 13 – Best Practices | Observability best practices module |
| 14 | Module 14 – Production Case Study | End-to-end production case study |

## Definition of Done for Each Module

Each module is considered complete when it includes:

- `README.md`
- `module.md`
- `slides/`
- `diagrams/`
- `examples/`
- `labs/`
- `quiz.md`
- `references.md`

The content must be suitable for a 1-hour training session and aligned with the course style guide.
